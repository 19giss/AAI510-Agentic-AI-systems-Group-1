## Acceptance Criteria

The VibeML agent should be able to:

- Accept a user request based on mood, activity, vibe, or location.
- Translate the user request into music-related features such as energy, tempo, danceability, and mood.
- Use Spotify/Kaggle data to search for matching songs.
- Recommend a small set of tester songs before creating the final playlist.
- Collect direct user feedback on tester songs.
- Adjust recommendations based on accepted or rejected songs.
- Store basic user preference notes when possible, such as disliked genres or preferred styles for certain activities.
- Generate a final playlist with a short explanation.
- Produce an evaluation trace showing the user input, tool calls, output, user feedback, and human review notes.
